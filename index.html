Predicting Best Sellers and Recommendations

This project aims to predict if a Book will make it onto the New York Times Best Sellers list,
given a review for the book. Then be able to reccomend a book based on the author and Average Rating.

Myself and The Project

Before Flatiron I had spent a year getting into coding more specifically web development. Although I could never dive into it due to a lack of direction and lack of connection with the material. This is a culmination of the skills I have learned here at this program as well as some of the web design skills that I came with.

For this proJect I wanted to know what the difference in reception was between a book that made it onto the New York Times Best-Seller list and one that didn’t. By doing this I hope to help both independent writers as well as larger publishing houses know what to look for in a completed work. I wanted to be able to tell you if A book is a likely bestseller given just one review.

Predictions

For the Prediction model in this project I was able to make an Api call to the New York times and find the corresponding books for the reviews I was able to get from an amazon review archive. Using this process I was able to get a total of 300,000 reviews for 15,000 books.

Once I had the reviews I removed the stop words and preprocced them using tfidf and Lemmatization. The results were somewhat surprising as what I took to be most important wasn’t the content of the book but whether it inspired us to share the experience with people around us. Drawing on these interpersonal connections seemed to be the most important thing about whether a book could make it onto the list or not.

After I had my clean data I then put it through different classification models. I started with Naive Bayes classifier Bernoulli, as I separated my data into a binary classification, and while it did well with a 71% accuracy score I wanted to try other models to see if I could get the score higher.

Next I tried Support Vector Classification, this one was surprising as because of the size of my dataset it was unable to converge and classified everything as a single class.

Lastly I tried Logistic Regression, This one performed the best with an accuracy score of 75% I believe because it was a wide and sparse dataset once it was vectorized also because of the binary classification.

Recs

For the recommendation system I was able to web scrape the data from goodreads, and needed to better familiarize myself with it.

I wanted to know what the most frequent books were as well as the most frequent authors and see if I could find a way to account for these so as to not always recommend them to everyone.

I also wanted to know what the distribution of the average rating for the books was.Once I knew this the next step was to know what the columns were like and if any of them were highly correlated.

Armed with all this knowledge I decided to focus on average rating while keeping the author in mind. I wanted to make a Nearest neighbors model and for this I needed to turn the average ratings into dummy variables, having turned them into only 5 different categories.

Finally I wanted to have an actual recommended list of books so I created a function that given a book title would find the 6 books closest to that one according to the fitted model and return that list.
